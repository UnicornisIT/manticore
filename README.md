# manticore

`manticore` — Flask-приложение для подготовки логинов абитуриентов и студентов Moodle.

Приложение помогает:

- загружать абитуриентов из Excel;
- автоматически формировать логины по договору;
- разделять абитуриентов по приемным кампаниям, например `2025` и `2026`;
- видеть дубли и конфликты логинов;
- вручную создавать и редактировать абитуриентов;
- отмечать, заполнена ли почта;
- переносить абитуриентов в список студентов;
- выгружать данные в Excel.

Проект рассчитан на запуск в Windows или Linux. Для обычной работы в локальной сети достаточно запустить приложение на одном компьютере и открыть его с других компьютеров по IP-адресу.

## Что не должно попадать в GitHub

В репозиторий не нужно добавлять:

- `.env` с реальными паролями и секретами;
- `.venv` или другие виртуальные окружения;
- `uploads/baze.db`;
- бэкапы базы;
- загруженные Excel/CSV-файлы;
- сгенерированные Excel-выгрузки;
- `__pycache__`;
- локальные копии проекта;
- архивы.

Это уже настроено в `.gitignore`.

В репозитории должна оставаться только пустая папка `uploads` с файлом `.gitkeep`, чтобы при первом запуске приложению было куда создать базу.

## Требования

Рекомендуется:

- Windows 10/11 или Linux;
- Python `3.11` 64-bit;
- доступ в интернет при первой установке зависимостей;
- браузер Chrome, Edge, Firefox или другой современный браузер.

Почему именно Python 3.11:

- старый Python 3.7 не подходит для текущих версий `pandas`;
- Python 3.11 стабилен и нормально работает с зависимостями проекта;
- установка через `start_one_click.bat` рассчитана именно на Python 3.11 x64.

Зависимости из `requirements.txt`:

```text
Flask>=2.2,<3.0
pandas>=2.0
openpyxl>=3.0
python-dotenv>=1.0
```

## Быстрый запуск в Windows в один клик

Для обычного пользователя Windows самый простой путь:

1. Скачайте или скопируйте папку проекта.
2. Откройте папку проекта.
3. Дважды нажмите:

```text
start_one_click.bat
```

Скрипт сделает следующее:

1. Проверит, установлен ли Python 3.11 64-bit.
2. Если Python 3.11 не найден, попробует установить его через `winget`.
3. Создаст виртуальное окружение `.venv`.
4. Обновит `pip`.
5. Установит зависимости из `requirements.txt`.
6. Создаст папку `uploads`, если ее нет.
7. Создаст `.env`, если его нет.
8. Запустит приложение.
9. Покажет адреса:

```text
http://127.0.0.1:5000
http://IP-КОМПЬЮТЕРА:5000
```

Первый адрес открывается на этом же компьютере.

Второй адрес нужен для других компьютеров в локальной сети.

### Если `winget` отсутствует

Если скрипт напишет, что `winget` не найден:

1. Установите Python 3.11 64-bit вручную:
   <https://www.python.org/downloads/windows/>
2. При установке обязательно включите галочку:

```text
Add Python to PATH
```

3. После установки снова запустите:

```text
start_one_click.bat
```

## Ручная установка в Windows

Если не хотите пользоваться one-click запуском, можно установить вручную.

### 1. Проверить Python

```powershell
python --version
```

Желательно:

```text
Python 3.11.x
```

Если выводится `Python 3.7`, `python37-32` или что-то похожее, установите Python 3.11 x64.

### 2. Создать виртуальное окружение

Из папки проекта:

```powershell
python -m venv .venv
```

### 3. Активировать окружение

```powershell
.\.venv\Scripts\Activate.ps1
```

Если PowerShell запрещает запуск скриптов, используйте:

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

Потом снова:

```powershell
.\.venv\Scripts\Activate.ps1
```

### 4. Установить зависимости

```powershell
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

### 5. Создать `.env`

Скопируйте `.env.example` в `.env`:

```powershell
copy .env.example .env
```

Отредактируйте `.env` при необходимости.

### 6. Запустить приложение

```powershell
python app.py
```

Откройте:

```text
http://127.0.0.1:5000
```

## Первый вход

При первом запуске приложение создает администратора:

```text
login: admin
password: значение ADMIN_DEFAULT_PASSWORD из .env
```

В `.env.example` пароль указан как пример. Для реальной работы обязательно задайте свой пароль в `.env`.

Пример:

```env
ADMIN_DEFAULT_PASSWORD=ChangeMeStrongPassword
```

Важно: если база уже создана, изменение `ADMIN_DEFAULT_PASSWORD` не изменит пароль существующего пользователя `admin`. Это значение используется только при создании пользователя в новой базе.

## Настройка `.env`

Пример `.env`:

```env
SECRET_KEY=replace-with-strong-secret-key-min-32-chars
ADMIN_DEFAULT_PASSWORD=123
UPLOAD_FOLDER=uploads
DB_FILENAME=baze.db
DEFAULT_CAMPAIGN_YEAR=2026
LEGACY_CAMPAIGN_YEAR=2025
FLASK_ENV=production
```

Описание:

| Переменная | Назначение |
| --- | --- |
| `SECRET_KEY` | Секретный ключ Flask. Нужен для сессий пользователей. |
| `ADMIN_DEFAULT_PASSWORD` | Пароль пользователя `admin` при первом создании базы. |
| `UPLOAD_FOLDER` | Папка, где лежит база и временные файлы. Обычно `uploads`. |
| `DB_FILENAME` | Имя файла базы SQLite. Обычно `baze.db`. |
| `DEFAULT_CAMPAIGN_YEAR` | Кампания по умолчанию для новых запусков, например `2026`. |
| `LEGACY_CAMPAIGN_YEAR` | Год для старых записей, если год невозможно определить. |
| `FLASK_ENV` | Окружение Flask. Для обычной работы можно оставить `production`. |

Не публикуйте `.env` в GitHub.

## Запуск в локальной сети

Приложение уже запускается так:

```python
app.run(host="0.0.0.0", port=5000, debug=True)
```

`host="0.0.0.0"` означает, что приложение доступно не только на этом компьютере, но и в локальной сети.

### Как открыть с другого компьютера

На компьютере, где запущено приложение, узнайте IP:

```powershell
ipconfig
```

Нужен IPv4-адрес, например:

```text
192.168.30.25
```

На другом компьютере откройте:

```text
http://192.168.30.25:5000
```

### Если сайт не открывается

Проверьте брандмауэр Windows.

Откройте PowerShell от администратора и выполните:

```powershell
New-NetFirewallRule -DisplayName "manticore Flask 5000" -Direction Inbound -Protocol TCP -LocalPort 5000 -Action Allow
```

Также проверьте:

- компьютер-сервер включен;
- приложение запущено;
- оба компьютера в одной сети;
- IP-адрес введен правильно;
- порт `5000` не занят другой программой.

## Как работает база данных

Используется SQLite.

По умолчанию база находится здесь:

```text
uploads/baze.db
```

Если базы нет, приложение создаст ее автоматически при запуске.

Основные таблицы:

| Таблица | Назначение |
| --- | --- |
| `abiturients` | Основной список абитуриентов. |
| `pending_duplicates` | Найденные дубли ФИО, ожидающие решения. |
| `login_conflicts` | Записи, где не удалось корректно сформировать логин или есть конфликт. |
| `students` | Список студентов для Moodle. |
| `students_duplicates` | Дубли студентов. |
| `users` | Пользователи приложения. |
| `groups` | Группы студентов. |

База не хранится в GitHub. Ее нужно переносить отдельно, если нужно сохранить реальные данные.

## Приемные кампании

Абитуриенты разделяются по полю:

```text
campaign_year
```

Например:

- `2025`
- `2026`

В интерфейсе есть переключатель приемной кампании.

Все основные действия выполняются внутри выбранной кампании:

- список абитуриентов;
- загрузка;
- ручное создание;
- редактирование;
- дубли;
- конфликты;
- очистка;
- миграция абитуриентов в студенты.

Это нужно, чтобы абитуриенты разных годов не смешивались.

## Формат файла абитуриентов

Файл должен быть Excel:

```text
.xlsx
```

Обязательные колонки:

```text
ФИО
Договор
```

Пример договора:

```text
2026-СД-0016-9И
2026-ФМ-0005-М
2026-ФМ-0006-2М
```

После загрузки приложение:

1. Разделяет ФИО на фамилию и имя-отчество.
2. Определяет год, специальность и базу из договора.
3. Генерирует логин.
4. Проверяет дубли.
5. Добавляет записи в выбранную приемную кампанию.
6. Формирует Excel-выгрузку с логинами.

## Формат файла студентов

Файл студентов загружается отдельно.

Формат:

```text
CSV
```

Колонки:

```text
username,password,email,firstname,lastname,cohort1
```

Шаблон находится здесь:

```text
static/students_template.csv
```

## Логика генерации логинов

Логин формируется из:

1. последних двух цифр года;
2. кода специальности;
3. кода базы;
4. порядкового номера.

Пример:

```text
2026-СД-0016-9И
```

Может дать логин вида:

```text
2639i001
```

### Коды специальностей

Коды задаются в `app.py`:

```python
spec_codes = {
    "ЛД": "1",
    "АД": "2",
    "СД": "3",
    "СтО": "4",
    "СтПр": "5",
    "СтП": "5",
    "ФМ": "6",
    "ЛабД": "7",
    "СтД": "8",
}
```

### Коды базы

Коды задаются в `app.py`:

```python
base_codes = {
    "2НМ": "inm",
    "2М": "im",
    "НМ": "nm",
    "М": "im",
    "11и": "11i",
    "9и": "9i",
    "11И": "11i",
    "9И": "9i",
    "11": "11",
    "9": "9",
}
```

Важно: длинные варианты проверяются раньше коротких. Поэтому `2М` не перепутается с `М`, а `НМ` не перепутается с `М`.

## Работа с дублями и конфликтами

### Дубли ФИО

Если при загрузке найден абитуриент с такой же фамилией, запись может попасть в раздел дублей.

Администратор может:

- одобрить дубль;
- отклонить дубль;
- отклонить все дубли текущей кампании.

### Конфликты логинов

В конфликты попадают записи, где:

- договор не удалось разобрать;
- логин уже занят;
- сформирован ошибочный префикс.

Администратор может открыть конфликт и назначить правильный логин вручную.

## Миграция старой базы

Приложение умеет подхватывать старую структуру базы.

Поддерживается случай, когда в старой базе абитуриенты лежали в таблице `students` с колонками:

```text
fio
dogovor
login
fam
imotch
email
comment
created_at
```

При запуске приложение:

1. Создает новую таблицу `abiturients`.
2. Переносит туда старых абитуриентов.
3. Присваивает им кампанию по году из договора.
4. Если год определить нельзя, ставит `LEGACY_CAMPAIGN_YEAR`.
5. Старую таблицу сохраняет как резервную:

```text
students_legacy_abiturients_backup
```

Таблица `students` после этого становится таблицей студентов Moodle.

Перед миграцией реальной базы рекомендуется вручную сделать копию:

```text
uploads/baze_backup_before_update.db
```

## Резервные копии

## Типовые ошибки

### `.venv` не создается

Если запуск остановился на создании виртуального окружения или папка `.venv` появилась не полностью:

1. Закройте окно запуска.
2. Удалите папку `.venv` из папки проекта.
3. Запустите `start_one_click.bat` еще раз.
4. Если `.venv` снова не появилась, откройте файл `install_run.log` в папке проекта. В нем будет точная причина ошибки.
5. Проверьте, что установлен Python 3.11 64-bit:

```powershell
py -3.11 --version
```

Ожидаемый вариант:

```text
Python 3.11.x
```

6. Запустите снова:

```text
start_one_click.bat
```

Если `py -3.11 --version` не работает, установите Python 3.11 64-bit с сайта:

```text
https://www.python.org/downloads/windows/
```

При установке включите галочку `Add Python to PATH`.

### `No module named dotenv`

Пакет `python-dotenv` не установлен.

Решение:

```powershell
python -m pip install -r requirements.txt
```

В актуальной версии приложения отсутствие `dotenv` не должно ломать запуск, но зависимости все равно лучше установить.

### `No module named flask`

Не установлены зависимости или запущен не тот Python.

Решение:

```powershell
python -m pip install -r requirements.txt
```

### `No matching distribution found for pandas>=2.0`

Скорее всего используется старый Python, например `Python 3.7 32-bit`.

Решение:

1. Установите Python 3.11 64-bit.
2. Создайте новое окружение.
3. Переустановите зависимости.

```powershell
python --version
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

### Приложение запускается, но база пустая

Проверьте `.env`:

```env
UPLOAD_FOLDER=uploads
DB_FILENAME=baze.db
```

Если `UPLOAD_FOLDER` указывает на другую папку, приложение будет искать базу там.

### Другие компьютеры не открывают сайт

Проверьте:

- приложение запущено;
- открыт правильный IP;
- открыт порт `5000`;
- Windows Firewall разрешает входящие подключения;
- компьютеры находятся в одной локальной сети.

### Не пускает в панель администратора

Проверьте в базе пользователя:

- `role` должен быть `admin`;
- `approved` должен быть `1`.

Кнопка панели администратора показывается пользователям с ролью `admin`.

```bash
sudo bash update.sh /srv/manticore
```

Проверка службы:

```bash
sudo systemctl status manticore
sudo journalctl -u manticore -f
```

Проверка Nginx:

```bash
sudo nginx -t
```

## Безопасность

Перед реальным использованием:

1. Измените пароль администратора.
2. Не публикуйте `.env`.
3. Не публикуйте `uploads/baze.db`.
4. Делайте резервные копии базы.
5. Не запускайте Flask debug-сервер в открытом интернете.
6. Для доступа из интернета используйте Nginx, HTTPS и нормальные учетные записи.
7. Если приложение работает только в локальной сети, ограничьте доступ средствами сети и брандмауэра.

Дополнительно см. [SECURITY.md](SECURITY.md).
