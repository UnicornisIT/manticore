# logins_moodle

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

Приложение на Flask для управления пользователями LMS Moodle. Обеспечивает обработку данных абитуриентов и студентов, автоматическую генерацию логинов, проверку дублей и управление пользователями.

## Функциональность

- 📋 Загрузка и обработка данных абитуриентов из Excel-файлов
- 🔑 Автоматическая генерация уникальных логинов на основе договора
- ⚠️ Обнаружение дублей и конфликтов логинов
- 👥 Управление пользователями (абитуриентами, студентами, персоналом)
- 🔐 Система ролей и авторизации
- 📊 Экспорт данных в Excel

## Требования

- Python 3.9+
- Flask >= 2.2
- pandas >= 2.0
- openpyxl >= 3.0

## Установка

1. Создайте виртуальное окружение:

```bash
python -m venv .venv
```

2. Активируйте окружение:

Windows PowerShell:

```powershell
.\.venv\Scripts\Activate.ps1
```

3. Установите зависимости:

```bash
pip install -r requirements.txt
```

## Запуск

```bash
python app.py
```

Откройте в браузере:

```
http://127.0.0.1:5000
```

## Настройки

- База данных создаётся автоматически в `uploads/baze.db`.
- Файлы загрузки сохраняются в папке `uploads`.
- При первом запуске создаётся пользователь с ролью `admin`.
- **Важно:** измените пароль администратора сразу после первого входа!

### Переменные окружения

- `SECRET_KEY` - секретный ключ Flask (генерируется случайно, если не установлен)
- `ADMIN_DEFAULT_PASSWORD` - пароль администратора по умолчанию
- `UPLOAD_FOLDER` - папка для загрузок
- `DB_FILENAME` - имя файла базы данных
- `FLASK_ENV` - окружение (development/production)

Скопируйте `.env.example` в `.env` и установите свои значения.

## Загрузка студентов

Для загрузки списка студентов используйте файл формата `CSV` с колонками:

`username, password, email, firstname, lastname, cohort1`

Пример шаблона для загрузки находится в `static/students_template.csv`.

## Развертывание на Linux

### 1. Подготовка

Скопируйте `.env.example` в `.env` и задайте свои значения:

```bash
cp .env.example .env
```

### 2. Развертывание

Сделайте скрипты исполняемыми и запустите их из корня проекта на сервере:

```bash
chmod +x deploy.sh update.sh
sudo ./deploy.sh /srv/logins_moodle_v3.5 example.com
```

Аргументы:

- Первый — путь для размещения приложения (по умолчанию `/srv/logins_moodle_v3.5`).
- Второй — доменное имя для Nginx (по умолчанию `example.com`).

### 3. Обновление приложения

После изменения кода на сервере выполните:

```bash
sudo bash update.sh /srv/logins_moodle_v3.5
```

### 4. Файлы конфигурации

- Шаблон systemd: `deploy/logins_moodle.service`
- Шаблон Nginx: `deploy/nginx.conf`

### 5. Проверка

```bash
sudo systemctl status logins_moodle
sudo journalctl -u logins_moodle -f
sudo nginx -t
```
